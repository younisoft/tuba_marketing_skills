# Task Router — Tuba Knowledge Platform

> **Part of:** [AI_KNOWLEDGE_PLATFORM.md](../AI_KNOWLEDGE_PLATFORM.md)
> **Purpose:** the exhaustive, systematic task-to-document lookup table — 100+ mappings. Where [../knowledge-graph/AI_INDEX.md](../knowledge-graph/AI_INDEX.md) gives full depth (required/optional/ignore/output/checklist) for ~20 major task categories, this document trades depth for breadth: fast, compact routing for the long tail of specific tasks.
> **Owner:** Knowledge Platform maintainer
> **Review frequency:** whenever a new task type or document is added

**Format:** Task → Read (in order, abbreviated paths — folder prefixes: `ai-sys`=advertising-system/, `bi`=brand-intelligence/, `bm`=brand-memory/, `kb`=knowledge-base/, `cp`=context-packs/, `ev`=evaluation/) → Output.

## Best Practices
- Use this table for fast lookup on a specific, narrow task; use [../knowledge-graph/AI_INDEX.md](../knowledge-graph/AI_INDEX.md) when the task is high-stakes and needs the fuller Required/Optional/Ignore/Output/Checklist structure
- Always still load [CLAUDE_SYSTEM_PROMPT.md](../CLAUDE_SYSTEM_PROMPT.md) regardless of which specific row is used — this table assumes that baseline is already active

## Common Mistakes
- Treating this table as exhaustive of all possible tasks — if a task genuinely isn't here or in AI_INDEX.md, reason from [../knowledge-graph/ONTOLOGY.md §8](../knowledge-graph/ONTOLOGY.md)'s retrieval model rather than skipping context entirely
- Reading only the first document in a row's list — the order matters, and later documents often contain the specific formulas/rules the earlier ones only introduce conceptually
