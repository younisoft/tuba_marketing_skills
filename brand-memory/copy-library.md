# Copy Library — Tuba

> **Layer:** Brand Memory (living)
> **Purpose:** the growing archive of actual, finished, approved copy instances Tuba has used — distinct from [brand-intelligence/headline-library.md](../brand-intelligence/headline-library.md) and [brand-intelligence/cta-library.md](../brand-intelligence/cta-library.md), which hold reusable *formulas*. This file holds the real, shipped sentences themselves, so nothing has to be rewritten from scratch and so repeated phrasing across campaigns can be spotted and avoided (or, if it's working, deliberately reused).
> **Owner:** Copy lead
> **Update trigger:** whenever a new piece of copy is finalized and approved (content-review-checklist.md passed)
> **Related documents:** [copywriting.md](../advertising-system/copywriting.md), [headline-library.md](../brand-intelligence/headline-library.md), [headline-performance.md](headline-performance.md)

## Best Practices
- Log the *exact final* text, not the draft — this library should reflect what actually shipped
- Before writing new copy for a recurring context (e.g., a monthly market report headline), check this library first for a proven starting point

## Common Mistakes
- Logging only headlines and skipping supporting copy/CTAs — the full asset's text has reuse value, not just the headline
- Letting this library become a place to copy-paste without adaptation — every reuse should still be checked against the current brief's specific audience and context (creative-brief-template.md)

## Future Expansion Notes
- Once volume grows, consider tagging entries by persona and funnel role for faster retrieval; cross-reference consistently high-performing entries into [brand-intelligence/headline-library.md](../brand-intelligence/headline-library.md) as new named formulas if a pattern proves durable
