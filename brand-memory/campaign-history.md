# Campaign History — Tuba

> **Layer:** Brand Memory (living)
> **Purpose:** the permanent, append-only database of every campaign Tuba has run — the single source of truth for "what have we actually done?"
> **Owner:** Marketing/Campaign Manager
> **Update trigger:** every campaign, logged at completion (advertising-playbook.md §1, Stage 10)
> **Related documents:** [advertising-playbook.md](../advertising-system/advertising-playbook.md), [campaign-playbook.md](../advertising-system/campaign-playbook.md), [winning-campaigns.md](winning-campaigns.md), [failed-campaigns.md](failed-campaigns.md)

**Status: no campaigns logged yet.** See [README.md §2](README.md) — this file will be populated as real campaigns complete. Do not fill entries with hypothetical data.

## Entry Template

Copy this block for every new campaign:

```markdown
## [Campaign Name] — [YYYY-MM-DD start] to [YYYY-MM-DD end]

**Objective:** [one sentence, from the original creative-brief-template.md]
**Audience:** [persona(s) targeted — link to messaging-framework.md or the relevant persona file]
**Campaign type:** [from campaign-playbook.md — seasonal/product/marketing-function]
**Creative summary:** [brief description + link to visual-library.md / copy-library.md assets used]
**Channels:** [platforms used]
**Budget:** [if paid — amount and allocation by channel]
**KPIs targeted:** [from the original brief, per advertising-playbook.md §5]
**Results:** [actual numbers against each KPI]
**Lessons:** [what this confirmed or challenged — link forward to lessons-learned.md if durable]
**Assets:** [links/references to final creative files]
**Related documents:** [links to the brief, the relevant campaign-playbook.md section, any persona files used]
**Logged by:** [name/role] on [date]
```

## Best Practices
- Log every campaign, including small routine ones — the value of this file compounds only with completeness
- Always link forward to winning-campaigns.md or failed-campaigns.md once a clear verdict is reached, rather than duplicating full analysis here

## Common Mistakes
- Waiting until "enough campaigns" have run to start logging — log from the very first one
- Logging only successful campaigns — see [failed-campaigns.md](failed-campaigns.md)'s explicit no-blame requirement

## Future Expansion Notes
- Once entry volume grows large, consider splitting this file by year or migrating to a structured database — but preserve the same field template for continuity
