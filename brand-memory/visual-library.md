## Organization

Organize entries by asset-library.md category (Photography, Icons, Templates, etc.):

```markdown
### [Category] — [Asset Name]

**File location:** [link/path]
**Used in:** [campaign(s), with links to campaign-history.md]
**Design Review score:** [from design-review-checklist.md, if scored]
**Reusable for:** [what future contexts this asset fits]
**Performance note:** [engagement/CTR if measured]
```

## Best Practices
- Log the final, shipped version of an asset, including its Design Review score — this creates a real track record of quality over time, not just a file dump
- Tag reusable templates clearly so future campaigns pull from proven layouts instead of rebuilding from scratch every time

## Common Mistakes
- Archiving raw/unused design drafts here — this library is for finished, approved, shipped work only
- Losing the link between an asset and the campaign it was built for, making performance attribution impossible later

## Future Expansion Notes
- As the library grows, track which templates get reused most often — high-reuse, high-scoring templates are candidates for formalizing as new default templates in [visual-system.md §9](../advertising-system/visual-system.md)
