## 3. How to Use Tags for Retrieval

A tag-based query should combine at least one **Audience** or **Content pillar** tag with one **Funnel role** or **Register** tag to narrow effectively — e.g., `#luxurybuyer + #propertyshowcase` should surface buyer-personas.md §2, luxury-framework.md, and campaign-playbook.md §2.6 without needing a full-text search.

## Best Practices
- Tag a new document at creation time, not retroactively — an untagged document is effectively invisible to tag-based retrieval
- Prefer 3-6 tags per document — over-tagging dilutes precision as much as under-tagging

## Common Mistakes
- Inventing a new tag ad hoc instead of checking §1's vocabulary first — tag sprawl defeats the purpose of a unified system
- Tagging by folder location rather than actual content (same mistake taxonomy.md §5 warns against)
