## 1. Large Master-Plan Developer

| Field | Detail |
|---|---|
| **Goals** | Sustained, large-scale buyer demand across a multi-phase project; brand-safe presentation matching their own scale and reputation |
| **Fears** | Being presented alongside lower-quality listings in a way that dilutes project prestige |
| **Pain points** | Needs structured, phase-aware presentation (not a single static listing) and qualified-demand data at volume |
| **Questions** | "How does Tuba present a project of our scale appropriately?" "What kind of buyer-intent data can you provide across phases?" |
| **Objections** | "We already have our own dedicated marketing/sales infrastructure" |
| **Emotional triggers** | Confidence, professional partnership (copywriting.md §5.2) |
| **Buying journey** *(partnership journey)* | Longer B2B sales cycle, multiple stakeholders, requires case studies and structured proposals |
| **Content preferences** | Data-forward partnership decks (visual-system.md §9.8), phased project-presentation examples |
| **Visual preferences** | Structured, professional, ground-level human imagery balanced with masterplan context (campaign-playbook.md §2.3's NHC-informed guidance) |
| **Best CTA** | Partnership inquiry → dedicated developer contact flow |

## 2. Boutique Developer

| Field | Detail |
|---|---|
| **Goals** | Standing out despite smaller scale; reaching genuinely interested buyers without a large marketing budget |
| **Fears** | Being lost among larger developer partners on the platform |
| **Pain points** | Limited internal marketing resources; needs the platform to carry more of the presentation/storytelling weight |
| **Questions** | "Can a smaller project get real visibility here, not just get buried?" |
| **Objections** | "Larger developers will always get priority placement" |
| **Emotional triggers** | Confidence, quiet pride in craft/distinctiveness |
| **Buying journey** | More relationship-driven, values direct contact with the Tuba partnership team |
| **Content preferences** | Project storytelling that emphasizes distinctiveness over scale (storytelling-framework.md) |
| **Visual preferences** | Craft-forward, specific (not generic masterplan renders) |
| **Best CTA** | Partnership inquiry |

## 3. Off-Plan Specialist Developer

| Field | Detail |
|---|---|
| **Goals** | Building buyer confidence in a property that doesn't exist yet; converting early-stage interest into deposits |
| **Fears** | Buyer hesitation/distrust specific to off-plan risk (market-psychology.md §2) |
| **Pain points** | Needs the platform to help carry credibility since buyers can't physically inspect the product |
| **Questions** | "How do you help us overcome off-plan trust hesitation?" |
| **Objections** | "Off-plan is a harder sell than ready-built — does this platform actually help with that specific problem?" |
| **Emotional triggers** | Confidence, trust-transfer (from platform credibility to project credibility) |
| **Buying journey** | Milestone-phased (announcement → deposit window → construction updates — campaign-playbook.md §2.3) |
| **Content preferences** | Construction-progress content, escrow/regulatory-clarity messaging (real-estate-terminology.md §3) |
| **Visual preferences** | Progress-transparency imagery, realistic renders paired with regulatory trust signals |
| **Best CTA** | Partnership inquiry → phased campaign structure |

## 4. Government-Linked / Institutional Developer

| Field | Detail |
|---|---|
| **Goals** | Supporting national housing/homeownership objectives (Vision 2030-aligned); scale and public trust |
| **Fears** | Any association that could appear inconsistent with public accountability standards |
| **Pain points** | Requires the most rigorous compliance/regulatory alignment among developer types |
| **Questions** | "How does this partnership align with regulatory and public-trust requirements?" |
| **Objections** | Procurement/approval processes likely slower and more formal than private-sector partners |
| **Emotional triggers** | Confidence, national-mission alignment (market-language.md register) |
| **Buying journey** | Longest, most formal B2B cycle; requires market-register communication (market-language.md), not consumer-register |
| **Content preferences** | Formal partnership proposals, regulatory-alignment documentation |
| **Visual preferences** | Most formal/restrained visual register in the developer set |
| **Best CTA** | Formal partnership inquiry (not a self-serve CTA) |

## Best Practices
- Match communication register to developer type: government-linked and large master-plan developers need market-register formality (market-language.md); boutique developers respond better to warmer, craft-focused storytelling
- Off-plan developer content should always foreground the specific trust mechanisms (escrow, regulatory compliance) that address the category's structural credibility gap (market-psychology.md §2)

## Common Mistakes
- Using identical partnership messaging across developer scale/type — a boutique developer and a large master-plan developer have different core anxieties (visibility vs. brand-safety)
- Presenting off-plan projects with the same trust-signal weight as ready-built listings — off-plan needs more, not equal, reassurance content
